# 포트폴리오 이력서 UI 고도화 디자인

## 1. 목적

마크다운 기반 포트폴리오 이력서를 채용 담당자가 빠르게 읽을 수 있는 전문 문서형 화면으로 개선함. 기존 콘텐츠 저장·렌더링 방식과 공개/관리자 경로는 변경하지 않음.

## 2. 적용 범위

| 구분 | 내용 |
|---|---|
| 대상 화면 | 공개 포트폴리오, 관리자 로그인, 포트폴리오 편집·미리보기 |
| 변경 파일 | `portal-web/app/static/css/portfolio.css` |
| 제외 범위 | 서버 구동, 스케줄러, 라우팅, 마크다운 저장 및 렌더링 로직 |

## 3. 디자인 방향

- 배경은 순백색 대신 옅은 청회색 바탕과 은은한 블루 계열 광원을 사용함.
- 이력서 본문은 백색 문서 카드로 유지하되, 테두리·그림자·상단 장식으로 문서의 밀도를 높임.
- 제목은 큰 크기와 타이트한 자간으로 강조하고, 섹션 제목은 네이비 포인트 라인 및 넉넉한 상단 여백으로 계층을 구분함.
- 본문·목록·인용문·표·코드·링크를 모두 읽기 쉬운 문서 요소로 정돈함.
- 모바일 화면에서는 카드 장식과 여백을 축소하되 가독성 및 터치 환경을 유지함.

## 4. 시각 토큰

| 항목 | 값 | 용도 |
|---|---|---|
| 주 텍스트 | `#162033` | 제목 및 본문 |
| 보조 텍스트 | `#64748b` | 설명 및 메타 정보 |
| 포인트 | `#1d4ed8` | 링크, 상단 라인, 버튼 |
| 배경 | `#edf2f8` ~ `#f7f9fc` | 페이지 바탕 |
| 카드 | `#ffffff` | 이력서 문서 및 관리 화면 |
| 구분선 | `#dbe4ef` | 섹션 및 표 경계 |

## 5. 구현 및 검증 기준

- 외부 폰트·이미지·JavaScript 의존성을 추가하지 않음.
- 마크다운이 생성하는 제목, 목록, 인용문, 표, 코드 블록, 링크가 모두 일관되게 표현되어야 함.
- 기존 600px 이하 반응형 범위를 유지 또는 개선해야 함.
- 포트폴리오 관련 테스트가 통과해야 함.
- 공개 및 관리자 화면이 동일한 CSS 파일을 사용하므로 관리 화면의 입력·버튼·미리보기 가독성도 유지해야 함.

## 6. 확인 필요 사항

- 실제 공개 이력서 마크다운 원문은 저장 볼륨에 있어 현재 작업공간에서 확인되지 않음. 일반 CommonMark 요소 기준으로 스타일을 적용함.
