# 뉴스 날짜·시간 표시 형식 설계

## 목표

프로젝트의 뉴스 화면에서 날짜와 시간을 한국시간(Asia/Seoul) 기준으로
`2026년 7월 10일 금요일 10:10:10` 형식으로 일관되게 표시한다.

## 범위

- `crawler-worker`의 뉴스 목록과 보관 뉴스 화면의 수집 시각 및 발행 시각
- 원본 ISO/RSS 시간값과 정렬·캐시 로직은 변경하지 않음
- 파싱할 수 없는 시간은 기존 원문을 그대로 표시해 화면 오류를 방지

## 설계

공통 Jinja 필터를 추가해 템플릿이 날짜 필드마다 직접 파싱하지 않도록 한다.
필터는 timezone-aware 또는 timezone-naive ISO 8601/RFC 822 값을 받아
Asia/Seoul로 변환하고, 한국어 요일을 포함한 `YYYY년 M월 D일 요일 HH:MM:SS`
문자열을 반환한다. `search.html`과 `saved.html`의 모든 시간 출력에 이 필터를
적용한다.

## 검증

- ISO 8601 UTC 값이 한국시간으로 변환되는지 테스트
- RFC 822 RSS 값이 한국어 날짜·시간으로 변환되는지 테스트
- 잘못된 값이 원문으로 안전하게 반환되는지 테스트
- 두 뉴스 템플릿이 공통 필터를 사용하는지 확인
