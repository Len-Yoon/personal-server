# 휘발성 데이터 자동 보존기간 정리 설계

## 목표

뉴스, 보안 로그, 백업처럼 계속 쌓이지만 장기 보존이 필요하지 않은 데이터를
보존기간이 지나면 자동으로 삭제한다. 정리는 하루 한 번 실행되며, 각 데이터의
보존기간은 환경변수로 조정할 수 있다.

## 기본 정책

- crawler-worker 뉴스 캐시: 7일
- Obsidian Investing 뉴스 Markdown: 30일
- 보안 로그: 30일
- 백업: 14일
- Docker 컨테이너 로그: 기존의 파일 크기/파일 개수 제한 유지

뉴스 캐시는 이미 `NEWS_RETENTION_DAYS`를 사용하므로 기존 만료 동작을 유지한다.
Obsidian 뉴스는 `YYYY-MM-DD.md` 파일명과 설정된 뉴스 디렉터리 내부라는 범위를
기준으로 오래된 파일만 삭제한다. 다른 Obsidian 파일은 건드리지 않는다.

## 실행 방식

기존 유지보수 스크립트에 `prune-news`, `prune-logs`,
`all` 명령을 제공하고, Windows 부트스트랩 데몬의 주기적 복구 루프에서 하루
한 번 `all`을 실행한다. 정리 실패가 서비스 복구를 중단시키지 않도록 각 대상은
독립적으로 처리하고 오류를 기록한다.

## 안전장치

- 날짜 형식이 `YYYY-MM-DD.md`와 일치하지 않는 Obsidian 파일은 삭제하지 않는다.
- 기준 시각은 파일 내용이 아니라 날짜 파일명으로 계산한다.
- 보존기간은 0일 미만을 허용하지 않는다.
- 삭제할 파일 목록을 테스트에서 검증하고, 실제 서비스 데이터에 대한 광범위한
  재귀 삭제는 사용하지 않는다.

## 검증

- 뉴스 캐시의 기존 7일 만료 테스트를 유지한다.
- Obsidian 뉴스 정리 테스트에서 오래된 날짜 파일은 삭제되고 최근 파일과
  날짜 형식이 아닌 파일은 유지되는지 확인한다.
- 로그/백업 정리 명령의 기존 동작과 `all` 명령을 테스트한다.
- 자동 정리 후에도 Windows 부트스트랩의 Docker/터널 복구 동작이 유지되는지
  기존 테스트로 확인한다.
