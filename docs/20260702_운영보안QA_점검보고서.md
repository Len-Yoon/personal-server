## 2. 핵심 요약

- 기본 Compose와 N100 override 구성이 분리되어 있으며, 운영 환경에서는 N100 override 사용이 적절함.
- 서비스들은 localhost 바인드 기준으로 운영되며, 공개 HTTPS는 Caddy가 담당하도록 구성되어 있음.
- 포털, 파일함, 메모 서비스, 시스템 상태 확인 흐름은 정상 동작 기준으로 정리되어 있음.
- Cloudflare DNS 및 외부 포트포워딩 여부에 따라 Caddy 방식 또는 Cloudflare Tunnel 방식 중 하나를 선택해야 함.

## 3. 상세 내용

### 3.1 점검 범위

| 구분 | 내용 | 비고 |
|---|---|---|
| 기본 Compose | `docker compose config` | 확인 완료 |
| N100 Compose | `docker compose -f docker-compose.yml -f docker-compose.n100.yml config` | 확인 완료 |
| 로컬 서비스 | `portal-web`, `system-agent`, `crawler-worker`, `youtube-memo`, `book-memo` | 구성 확인 |
| 공개 HTTPS | Caddy + Cloudflare DNS challenge | 구성 확인 |
| 대체 공개 경로 | Cloudflare Tunnel | 외부 포트포워딩 불가 시 사용 |

### 3.2 구성 확인 결과

| 구분 | 결과 | 비고 |
|---|---|---|
| 기본 Compose | 일치 | 개발용 공개 포트 및 bind mount 구성 확인 |
| N100 Compose | 일치 | localhost 바인드, 자원 제한, 보안 옵션 적용 확인 |
| 메인 진입점 | 일치 | `len.pe.kr` 기준으로 정리함 |
| 호환용 별칭 | 확인 필요 | `portal.len.pe.kr`은 호환용 별칭으로 유지 가능함 |
| 공개 HTTPS | 확인 필요 | `CADDY_EMAIL`, `CLOUDFLARE_API_TOKEN` 설정 필요함 |
| Cloudflare Tunnel | 일치 | 외부 포트포워딩이 어려운 환경의 대안으로 유지함 |

### 3.3 검토 메모

- `portal-web`은 `/`, `/files`, `/admin/status`를 함께 처리하므로 공개 진입점과 내부 라우팅을 함께 고려해야 함.
- `crawler-worker`는 뉴스 검색/요약 기능에 사용되므로 기본 운영 스택에 포함하는 편이 적절함.
- `system-agent`는 운영 상태 확인용으로 비공개 운영을 권장함.
- 공개 HTTPS를 사용할 경우 Cloudflare DNS 설정과 서버의 `80`/`443` 접근 가능 여부를 함께 확인해야 함.

## 5. 확인 필요 사항

- N100 PC에서 실제로 `80`/`443` 포트가 열려 있는지 확인 필요함.
- Cloudflare DNS의 `len.pe.kr` A 레코드가 N100 공인 IP를 가리키는지 확인 필요함.
- `CADDY_EMAIL`, `CLOUDFLARE_API_TOKEN` 값이 `.env`에 정상 입력되어 있는지 확인 필요함.
- `portal.len.pe.kr`을 호환용 별칭으로 유지할지 최종 결정 필요함.

## 6. 후속 조치

- N100 PC에서 최신 변경사항 pull 후 `docker compose -f docker-compose.yml -f docker-compose.n100.yml up -d --build` 실행 필요함.
- Caddy 로그와 컨테이너 상태를 확인해 초기 인증서 발급 여부 점검 필요함.
- 외부 포트포워딩이 불가능하면 Cloudflare Tunnel 구성으로 전환 필요함.
